1概要 — 何を / なぜ market-free

本システムは、JRA中央平地レースの各馬について p_ls = その馬が複勝圏（3着内）に来る確率 を予測する深層モデル群。 GNN（グラフ的関係推論）と呼ぶが実体は 過去30走を系列として読む Transformer + レース内相互注意 で、 「この馬の過去走」「その時の相手」「相手の能力」を多段で参照する（user の言う“数珠繋ぎ”=multi-hop relational reasoning）。

★市場フリー（market-free）の核心★
当日の単勝オッズ等の市場情報をモデル入力に一切入れない。能力・血統・相手成績・条件替わり・調教だけから独力で確率を導く。 理由：オッズに寄せると「市場の劣化コピー」にしかならず勝ち筋が消えるため。オッズは評価と買い目選択の段でのみ使う（帯の線引き等）。 トレードオフ：順位精度は市場（AUC≈0.82）より弱い（自力0.73級）が、市場と独立=食い違いがエッジ源になる。

同一アーキテクチャをオッズ帯別に4つ学習：BASE 全馬HON 1-10倍 MID 10-30倍ANA 30-100倍（穴）。 各GNNは担当帯の「好走の理屈」に特化する（後述§7）。

2システム全体図（データの流れ）

JV-Link → 前処理DB(nn_core) → 入力テンソル組立(動的join+leak-free標準化) → 4GNN → p_ls → ダッシュボード

3データパイプライン（nn_core 250列）

build_nn_v5.py が JV-Link 由来の生データから学習可能DB nn_core_full.parquet （808,329 インスタンス × 250 列、全列 leak-free 検証済）を構築。Phase A–D ＋ 後処理4本で完成。

段階内容主な出力
Phase Aparse/load（RACE/SE/HR/UM/O1…/調教 を解釈）+ オッズ long 化race/entry/odds
Phase B本馬の過去成績 123列（着順/走破残差/上がり/勝率…を groupby(horse).shift(1).expanding で過去集計＝leak-free）horse_perf 123列
Phase Cコース・調教の紐付け（最終追い切り 坂路7区間/ウッド終い3区間）train_link
Phase Dnn_core 組立（過去30走ポインタ方式B、血統sire/dam/母父100%、T1-T5ターゲット）nn_core 234列
後処理①build_collateral_cw：相手の競争成績をクラス加重log賞金で（collcw_* 6列, 約21分）+6列
後処理②build_v5_judge_change：状況変化（距離/クラス/馬場替わり chg_* 3列）+3列
後処理③build_v5_track_pedigree：馬場傾向 + 血統適性（track_maeno/ped_* 等）+列
後処理④merge_v5_enhancements：全結合 → 250列 完全版nn_core 250列
リーク防止の鉄則（RULE-2）：レース後にしか分からない量（着順・走破時間・上がり・通過順位）は生で入れない。 必ず groupby(horse_id).shift(1) で過去集計化してから入力にする。新規 raw 列は投入前に「レース前/後」を判定。

4入力テンソル（過去30走の動的join）

nn_join.py が、各予測インスタンスに対し過去30走を動的に集めてテンソル化（方式B：集約table→batch時gather、RAM約1GB）。 1走あたり「本馬成績」「同走18頭（相手）」「ラップ/スピード指数」「調教」「6馬券オッズ履歴」を保持。

テンソル形状内容
past_self[B, 30, 142]各過去走の本馬成績（perf120 + 結果12 + 文脈10）
past_opp[B, 30, 18, 132]各過去走の同走18頭の成績（相手の強さ=数珠繋ぎ1hop）
past_lap[B, 30, 25]各過去走の生ラップ（レース先頭基準）
past_sf / past_train[B,30,3] / [B,30,14]スピード指数 / 調教（坂路7+ウッド3区間）
past_odds[B, 30, 18, 3]過去走の単複オッズ履歴（※当日のオッズではない）
past_mask / recency[B, 30]有効走mask（新馬=0走対応）/ 各走「何日前」
cur_num[B, 37+欠損ind]現走の条件（距離/馬場/斤量/年齢…）+ 本馬過去集計 + 相手成績 + 変化flag

leak-free 標準化（nn_folds.py, RULE-2）：各 fold で train（年<cutoff）だけで平均/分散を算出して全適用。 欠損は標準化後0（中立）＋「欠損indicator」を concat してNNに明示。語彙（horse/sire/…）も train のみ、未知は idx0(UNK)。

5モデル内部アーキテクチャ

過去走エンコーダ → 言い訳ゲート → 6層Transformer → 現走queryでcross-attention → レース内attn×2 → 各ヘッド

因子分解embedding + FiLM条件変調

馬を horse_id(32) + sire(16) + dam(16) + trainer(16) + jockey(16) + 母父(sire空間共有16) の因子分解埋め込みで表現（rare→UNK=0）。 現走の条件ベクトルから FiLM（scale,shift）を生成し、能力表現を当該条件で連続変調（セルを切らずに条件別能力を表現）。

調教/SFMLP(25+3+14)→24

65つの工夫（穴を読む仕掛け）

#仕掛け狙い / 実装
②条件適性プロファイルz→適性ベクトル(k=8, Tanh)、今日の条件→条件ベクトル、その一致(match=apt×cond)をヘッドへ。「この条件で走る馬か」を表現
③言い訳ゲート各過去走に「今日への関連度」をsigmoidで学習し過去走を再重み付け。今日と無関係な凡走を自動割引（適性awareに：今日の調教/装備EXTRAをqueryへ注入）
時間Temporal Encoding各過去走の「何日前」を Time2Vec（線形trend+多周波sin）で表現し過去走に加算。直近/休み明け/季節を区別
④レース内attention ×2同一レースの出走馬どうしを2層で相互注意。「場の中での相対的強さ」を深く推論
⑤+①マルチタスク + 自己教師主3ヘッド(top3/top5/win)に加え補助(脚質/ペース/SF)、さらに SSL：過去走15%を隠して復元（BERT流, ラベル不要で全データ事前学習）

74モデルの帯特化（BASE/HON/MID/ANA）

同じアーキテクチャをオッズ帯別の学習データで4つ学習。各GNNが見る“好走の理屈”は帯で異なる（gnn_logic.py で代理解釈）。

モデル担当オッズ帯主に見ているもの確信度の信頼性
BASE全馬（土台）過去実績ほぼ一本（過去平均着順の重視度ほぼ100）最上位の確信は実複勝率59%＝高信頼
HON1–10倍（本命）実績 + 小頭数 + 若さ上位の本命は複勝率69%
MID10–30倍（中穴）言い訳・上昇余地（クラス/距離/芝ダ替わり, 相手強, 前走不利）一番ガタつく（信頼やや低）
ANA30–100倍（穴）条件替わり・隠れた素質（若さ/フレッシュ/距離替わり/前走の脚/斤量/クッション）上位5-10%のみ有効・市場をわずかに超える
学習対象は平地のみ（障害=track_code≥50 を除外）。ANAは穴=30≤オッズ<100倍の好走（is_ls）に特化。 4モデルの“合議”（何モデルが本命視したか）が最も頑健なシグナルで、ダッシュボードの◎本命/⚠危険判定に使う。

8ターゲットと損失関数

主タスク=好走3種を focal BCE（クラス不均衡対策, γ=1.5）で学習。p_ls は top3 ヘッドの sigmoid。

L_hr = Σ_k focal_BCE(hr_k, y_k ; pos_weight_k, γ=1.5), k∈{top3, top5, win}
pos_weight = top3:13 / top5:5 / win:20 （正例が少ない=正例損失を増幅）

補助タスク（マルチタスク, 合計重み0.2）：スピード指数 t1 / 数珠繋ぎ t3（相手の次走speed残差=multi-hop supervise）/ 展開 t5（前残り度・直線利得）等を MSE。NaNはmask。各targetは train-fold統計でz化（スケール統一）。

自己教師(SSL)：過去走の15%をマスクし、残り+今日条件から隠した走の(着順/speed/上がり)を復元。ラベル不要で全データ事前学習→本学習をwarm-start。

数珠繋ぎ（multi-hop）：馬A→Aの過去走の同走馬B（1hop, 明示保持）→Bの次走（2hop, embedding経由）… を Tier3 ターゲット opp_level_correction = 本馬speed − 同走馬の次走speed平均 でNN supervision として直接学習。

10市場フリー原則と評価

  • 入力に市場を入れない：当日オッズ・人気をモデル入力から完全排除（過去オッズ履歴は可）。
  • 主指標=複勝圏 lift@10%：確信度上位10%の複勝率/全体複勝率。ANAで 1.34（6/6 fold で>1）。
  • 到達点（正直）：実力は本物で安定（毎年ランダム超・控除率の約6割回収）だが、公開データの複勝はこの辺が天井＝控除率を持続的に破る単独馬券は無い。
  • 使い道：勝つための馬券ではなく、レースの堅さ判定・本命/穴の抽出・危険な人気馬の検出（市場の過剰人気を突く）に有効。
本モデルは「市場を超える予測器」ではなく「市場と独立な良い確率推定器」。 ダッシュボードの推奨は🏆全モデル一致の本命単勝のみ＝堅さの提示であり、利益保証ではない。
